# Zarr test fixtures

All stores except `2d_scalar_v3.zarr` (see its own section) are
generated by `test/data/generate_zarr_fixtures.py` with the fridom
framework2 zarr `Writer` (tensorstore sink, zarr format v2). Fully
deterministic: every variable holds `cos(it * 0.5) * f(coords)` at
iteration `it` (float64), with `f` given per store below. Coordinates
are the grid evaluation nodes of uniform interval meshes.

## Writer format notes (apply to every Writer-generated v2 store)

- **zarr v2**, one directory per array, `dimension_separator: "."`.
- **No `.zmetadata`** — the Writer never consolidates metadata.
  (Python needs `xr.open_zarr(path, consolidated=False)`.)
- Compressor on every array: `blosc` (`cname: "lz4"`, `clevel: 5`,
  `shuffle: -1`, `blocksize: 0`). No filters.
- `fill_value: null` in every `.zarray`.
- Data variables are `<f8` (float64), shaped `(time, *spatial)`;
  `time` is `<f8`, `iteration` is `<i8`. Default chunking: 1 time
  step per chunk, spatial dims whole.
- Dimension names come from `_ARRAY_DIMENSIONS` in each array's
  `.zattrs` (xarray convention).
- Every data variable carries a `coordinates: "iteration"` attr — CF
  auxiliary-coordinate promotion of the `iteration(time)` array
  (`long_name: "iteration"`).
- Staggered coordinates carry the comodo attr
  `c_grid_axis_shift: 0.5` (`x_right`, `y_right`, `y_outer`,
  `z_outer`); cell-center coords (`x`, `y`, `z`) have no shift attr.
- Time axis: `long_name: "time"`, `standard_name: "time"`,
  `axis: "T"`; `units` is plain `"seconds"` without a calendar
  anchor, or `"seconds since <ISO datetime>"` plus
  `calendar: "proleptic_gregorian"` when the clock has a start date.
- Global attrs on every store: `Conventions: "CF-1.10"`,
  `fridom_version`, plus a per-store `title`.

Ground-truth indices below are **0-based, C-order, time first**
(Python convention; add 1 per axis for Julia, and note the axis
order may appear reversed in column-major readers).

## 2d_scalar.zarr

2D cell-centered scalar with an animation-length time axis.

- Dims: `time: 5`, `x: 16`, `y: 16`.
- Coords: `x`, `y` in [0.03125, 0.96875] (cell centers of [0,1),
  both periodic); `time` = [0.0, 0.5, 1.0, 1.5, 2.0] (units
  `"seconds"`, no calendar); `iteration` = [0..4].
- Variable: `temp(time, x, y)` — `long_name: "Temperature"`,
  `units: "K"`. `f = sin(2πx)·cos(2πy)`.
- Chunks: `[1, 16, 16]`. No `.zmetadata`.
- Ground truth:
  - `temp[0, 0, 0]   = 0.19134171618254486`
  - `temp[2, 8, 8]   = 0.1033823704621962`
  - `temp[4, 15, 15] = 0.07962624988886739`

## time_chunked_append.zarr

Time-chunked store with a CF calendar time axis, written in two
Writer sessions (`mode="w"` create, then `mode="a"` append) — the
5-step axis spans both segments and the final time chunk is partial.

- Dims: `time: 5`, `x: 8`, `y: 8`.
- Coords: `x`, `y` in [0.0625, 0.9375] (centers of [0,1),
  periodic); `time` raw values [0.0, 0.5, 1.0, 1.5, 2.0] with
  `units: "seconds since 2020-01-01T00:00:00"` (whole-second
  reference date since the fridom Writer CF-units fix) and
  `calendar: "proleptic_gregorian"` (decodes to datetimes
  2020-01-01T00:00:00 … 00:00:02); `iteration` = [0..4].
- Variable: `c(time, x, y)` — "Tracer concentration",
  `units: "kg/m3"`. `f = exp(−((x−0.5)² + (y−0.5)²)/0.02)`.
- Chunks: `[2, 8, 8]` (time chunk 2; 5 steps → 3 chunks, last
  partial). No `.zmetadata`.
- Ground truth:
  - `c[0, 0, 0] = 4.867793902108199e-09`
  - `c[2, 4, 4] = 0.3655868273096103`
  - `c[4, 7, 7] = -2.0257170333257972e-09`

## 2d_scalar_v3.zarr (zarr format v3)

The only **zarr v3** store — generated by
`test/data/generate_zarr_v3_fixture.py` (plain xarray
`to_zarr(..., zarr_format=3)`, Python `zarr>=3`), *not* by the fridom
Writer, which cannot emit v3. Layout differs from the v2 stores
accordingly: `zarr.json` metadata documents (group root and one per
array), **no** `.zgroup`/`.zarray`/`.zattrs`/`.zmetadata`, chunk keys
under `c/` with `/` separator, `bytes` + `zstd` (level 0) codecs,
`fill_value: NaN` on the data variable, dimension names in the native
v3 `dimension_names` field instead of `_ARRAY_DIMENSIONS`.

As of Zarr.jl 0.9 / ZarrDatasets 0.1.5 the Julia stack cannot read
v3; CDFViewer detects the root `zarr.json` and raises a clear
"zarr v3 not yet supported" error (asserted in
`test/test_zarr_fixtures.jl`). Zarr.jl 0.10 can read this store, but
ZarrDatasets caps Zarr at 0.9.

- Dims: `time: 3`, `x: 8`, `y: 8` (variable order time, x, y).
- Coords: `x`, `y` = cell centers `(i + 0.5)/8` in
  [0.0625, 0.9375]; `time` is datetime64 — raw int64 [0, 30, 60]
  with `units: "minutes since 2020-01-01 00:00:00"`,
  `calendar: "proleptic_gregorian"` (decodes to 2020-01-01T00:00,
  00:30, 01:00).
- Variable: `temp(time, x, y)` float64 —
  `long_name: "Temperature"`, `units: "K"`;
  `temp[it, ix, iy] = cos(it·0.5)·sin(2πx)·cos(2πy)`.
- Chunks: `[3, 8, 8]` (single chunk).
- Global attrs: `Conventions: "CF-1.10"`,
  `title: "zarr v3 fixture: 2D scalar"`.
- Ground truth (0-based, time-first):
  - `temp[0, 0, 0] = 0.3535533905932738`
  - `temp[1, 4, 4] = 0.31027229028187275`
  - `temp[2, 7, 7] = -0.19102571218504516`

## Variants not produced

- **float32 data**: fridom runs JAX in x64 mode; the Writer stores
  fields at their native dtype, so every fixture is float64. There
  is no supported knob to make the real Writer emit float32.
- **consolidated metadata**: the Writer never writes `.zmetadata`,
  and the v3 fixture is written with `consolidated=False`, so no
  fixture has it.

## Regenerating

```
uv run --project <fridom checkout> python \
    test/data/generate_zarr_fixtures.py

cd test/data && uv run --with "zarr>=3" --with xarray \
    --with numpy --with pandas python generate_zarr_v3_fixture.py
```

Note: regenerating deletes and recreates the `.zarr` stores but does
not touch this manifest.
